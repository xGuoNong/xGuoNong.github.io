1 Git概述

Git是一个免费的、开源的分布式版本控制系统（Distributed Version Control System，简称 DVCS），分为两种类型的仓库：本地仓库和远程仓库。

a. 集中式版本控制工具
集中式版本控制系统，版本库是集中存放在中央服务器，团队和个人工作时从中央服务器下载代码，必须联局域网或互联网才能工作，个人修改后再提交到中央版本库。

b. 分布式版本控制工具
分布式版本控制系统没有中央服务器，每个人的电脑上都是一个完整的版本库，这样工作的时候，无需联网，多人协作时只需要将各自的修改推送给对方，就能互相看到对方的修改了。

2. Git的安装

Git官网：https://git-scm.com/download

1) Git选项配置，默认选项，点击“Next”

  1. 选择一个Vim编辑器，点击“Next”

  2. 是否修改初始化分支的名字，选择“Let Git decide”

  3. 是否修改环境变量，选择“Use Git from Git Bash only”

  4. 选择SSH可执行文件,勾选“use bundled OpenSSH”

  5. 勾选“Use the OpenSSL library”

  6. 勾选“Checkout Windows-style， commit Unix-style line endings”

  7. 勾选“Use MinITY (the default terminal of MsYs2)”

  8. 选择默认的“git pull”行为

  9. 选择凭据管理器

  10. 其他配置

  11. 配置实验室功能（不要勾选），点击“Install”，等待安装完成

  12. 安装完成后，在桌面单击鼠标右键选择“Git Bash Here”。

  13. 在弹出的对话框中，输入 git --version ,出现版本号，则表明安装成功

3 GIt常用命令

命令名称 作用
git config —global user.name 用户名 设置用户签名
git config —global user.email 邮箱 设置用户签名
git init 初始化本地库
git status 查看本地库状态
git add 文件名 添加到暂存区
git commit -m “日志信息” 文件名 提交到本地库
git reflog 查看历史记录
git reset —hard 版本号 版本穿梭

Git首次安装必须设置用户签名，其作用是区分不同操作者身份，否则后期无法提交代码。此处设置的用户签名和将来登录GitHub的账号无任关系。
